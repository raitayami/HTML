Semantics HTML

Introduction to Semantic HTML

Semantic elements provide information about the content between the opening and closing tags

Rather than using just div and span, using HTML elements based on their meaning is SEMANTIC HTML

Using a header element instead of a div element, ir povides context as to what information is inside of the opening/closing tag

Why SEMANTIC HTML

  • Accessbility. Screen readers and browsers will be able to interpret the code better
  • SEO. Improves Search Engine Optimization
  • Easy to understand

Think of a shop with no signs on the aisles. That is non semantic

A shop with signs for each aisle make its alot easier to find items you need

Header and Nav

A header tag is usually a container consisting of navigational links or introductory content containing h1 to h6

Now imagine having a div with an id of header. Compare them two. Using header allows easier read, identfiication

Using div provides no details

Using a nav, it defines a block of navigation. Nav can also be used inside a header element

Main and footer

The main element is used to encapsulate teh dominant content within a webpage

This tag is separate from teh footer and the nav since these elments don't contain the principal content

Using main instead of div element , screen reader and browsers are better able to identify that whatever is inside of the tag, is the bulk of the content

Information within the footer tag are details such as contact information, copyright, terms of use

The footer tag is seperate from the main element

Artcle and Section

These two elements can go in the body, in the main

Section defines elements in a document, such as chapters, headings or any other area of the document with the same theme

Content about the same theme can go under a single section

A website's homepage can be split into sections such as introduction, news, and contact information

Article element holds content that makes sense on its own

Article element can hold content such as artcles, blogs, comments

An article element can help a screen reader undersands where the article content begins and ends.

A section can be placed in an artcle element depend on context. And an article can be placed inside a section element

Aside element

This is used to mark additional information that can enhance another element

It can be used alongside article, or section.

Comments, quotes, bibliographies can go in here

Figuure and Figcaption

The figure element is used to encapulate media such as an image, illustration, diagram, etc...

You can insert your image tag in here

The figcaption is an element you can use to describe the media in the figure tag

The figcaption element goes inside the figure element

Audio and attributes

The audio element is used to embed audio content into a document

You have to create an audio tag, then insert a source tag inside it. Source is a self closing tag with src, controls and type as the attribute

Video and Embed
